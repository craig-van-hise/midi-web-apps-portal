# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
Refactor and optimize the `StepSequencer` layout control parameters:
1. **Event Re-binding:** Move the selection interaction from `onClick` to `onPointerDown` to prevent latency and double-trigger artifacts.
2. **Bi-directional Editing:** Establish a real-time sync wrapper. When a chord step is selected on the timeline, any manual inputs or transformations executed on the main `NotationCanvas` or virtual/physical `Keyboard` must instantly update that specific slot's sequence data and chord symbol badge.
3. **Blank State Insertion:** Allow users to select empty/blank timeline steps. Selecting a blank card clears the main notation grid, preparing it to bind fresh notes directly into that slot from a blank state.

## 2. Technical Decisions & Dependencies
* **Target File:** `src/plugins/chord-notator/components/StepSequencer.tsx`
* **Ref Lifecycles:** Introduce `selectedStepRef` inside the core `useEffect` block to prevent stale closure captures from checking old state cycles during asynchronous MIDI event polling.
* **State Injection Pattern:** Inside the `MIDI_MESSAGE_RECEIVED` sniffer, check if `selectedStepRef.current !== null`. If true, automatically compute the real-time `getChordSymbol` matrix for incoming values and apply them directly back into the sequence slot.
