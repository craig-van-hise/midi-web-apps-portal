# Product Requirements Prompt (PRP) — 🔵 Active

## 1. Project Context & Objectives
**Context:** In the `chord-notator` plugin, the previous "over-tuck" strategy for the transformations drawer successfully hid the bottom corners, but pushed the drawer so far up that its top edge now visibly peeks out from behind the top of the keyboard card. 
**Objective:** Abandon the `translateY` z-index "slide behind" trick. Refactor the drawer to use a zero-height collapse strategy (`height: 0` + `overflow: hidden`) so it physically disappears from the DOM flow rather than hiding behind another element. This will permanently prevent both bottom-gap and top-peek UI bugs.

## 2. Technical Decisions & Dependencies
* **Architecture Shift:** Instead of translating the drawer up the Y-axis behind the keyboard, we will wrap the main drawer body in a `motion.div` that animates its `height` between `"auto"` and `0px`, with `overflow: hidden`. 
* **Tab Placement:** The chevron toggle tab must remain completely outside of this collapsing wrapper so it is always visible.
* **Framer Motion (`motion`):** Animating height to `auto` is natively supported by Framer Motion and is the industry standard for accordion/drawer patterns.

## 3. Implementation Phases

### Phase 1: Structural Refactor (Height Collapse Strategy) — ✅ Completed
* **Objective:** Restructure the DOM so the drawer collapses into itself rather than sliding behind the keyboard.
* **Instructions:**
  1. Locate the component containing the transformations drawer.
  2. Remove the negative `translateY` (or `y: "-100%"`) logic from the Framer Motion `variants`.
  3. Ensure the z-index of the drawer is normal (it should sit below the keyboard in the standard document flow, not absolutely positioned behind it).
  4. Wrap the *main body* of the drawer (the white card containing the buttons) in a `motion.div`. Apply `overflow: hidden` to this wrapper.
  5. Update the Framer Motion variants on this wrapper to: 
     * `open: { height: "auto", opacity: 1 }`
     * `closed: { height: 0, opacity: 0 }`
* **TDD Checkpoint:**
  * **Test Case 1:** *Given* the drawer state is set to closed, *Then* the wrapper's height resolves to 0px and its contents are entirely invisible due to `overflow: hidden`.
  * **Status Update:** "Transition Phase status to 🔵 Active. Refactor the wrapper and variants. Once applied, update status to ✅ Completed."

### Phase 2: Chevron Tab Re-anchoring — ✅ Completed
* **Objective:** Ensure the chevron tab sits directly flush against the bottom of the keyboard when the drawer body collapses to 0 height.
* **Instructions:**
  1. Place the chevron tab element *outside* and *below* the new `overflow-hidden` collapsing wrapper in the JSX structure.
  2. Because the wrapper collapses to 0px, the tab will naturally slide up to sit directly underneath the keyboard container.
  3. Remove any lingering absolute positioning, negative bottom margins, or arbitrary offsets from the tab. Let standard CSS flexbox/block layout stack it flush against the collapsed wrapper.
* **TDD Checkpoint:**
  * **Test Case 1:** *Given* the drawer is closed, *Then* the chevron tab sits immediately adjacent to the bottom edge of the keyboard card, driven by standard document flow rather than absolute coordinates.
  * **Status Update:** "Transition Phase status to 🔵 Active. Restructure the tab's CSS. Once applied, update status to ✅ Completed."
