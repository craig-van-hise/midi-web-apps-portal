# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The newly implemented Floating Pill `KeySignatureSelector` has a minor layout regression: the "ROOT NOTE" micro-label is wrapping onto two lines. This breaks the horizontal alignment with the adjacent "SCALE TYPE" section. The objective is to enforce strict single-line rendering for these micro-labels.

## 2. Technical Decisions & Architecture
* **CSS / Tailwind Strategy:** The wrapping occurs because the intrinsic width of the `E` (or other single-character root notes) in the select box below is smaller than the width of the heavily tracked "ROOT NOTE" text. We will apply Tailwind's `whitespace-nowrap` utility directly to the `span` elements containing the micro-labels. This forces the flex-column container to expand to the width of the text, preventing any line breaks.

## 3. Implementation Phases

### Phase 1: Enforce Whitespace Rules on Micro-Labels — ✅ Completed
* **Objective:** Prevent the micro-labels from wrapping onto multiple lines.
* **Instructions:**
  1. Open `src/plugins/chord-notator/components/KeySignatureSelector.tsx`.
  2. Locate the `span` element containing the text "Root Note" (or "ROOT NOTE").
  3. Append `whitespace-nowrap` to its `className` string. The updated class should look exactly like this:
