
# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The `StepSequencer` noteheads and accidentals are rendering as grey instead of black in light mode. This is caused by inline `color` styles overriding the CSS hierarchy, breaking Tailwind's native dark/light mode `text-black dark:text-gray-300` utility classes. The objective is to replace the inline style color variables with pure conditional Tailwind class strings.

## 2. Technical Decisions & Dependencies
* **Target File:** `src/plugins/chord-notator/components/StepSequencer.tsx`
* **Logic Specification:**
  * Define `const isSelected = selectedStep === idx;` at the top of the note render loop.
  * Derive text color via Tailwind: `const textCol = isSelected ? 'text-[#aa3bff]' : 'text-black dark:text-gray-300';`
  * Derive background color for ledger lines via Tailwind: `const bgCol = isSelected ? 'bg-[#aa3bff]' : 'bg-black dark:bg-gray-400';`
  * Remove `color` and `backgroundColor` properties from the inline `style` objects.
