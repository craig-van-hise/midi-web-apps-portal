# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The `NotationCanvas` layout engine's Crossover Inhibition rule successfully prevents standard chords from visually invading the opposite staff. However, for massive chords spanning 3+ octaves (e.g., C4 to C8), the inhibition blocks octavation entirely, resulting in infinite ledger lines that overflow the physical boundaries of the canvas. The objective is to implement an **Extreme Bypass** rule: `8va`/`8vb` shifts will continue to respect crossover limits, but extreme `15ma`/`15mb` shifts will forcefully bypass the limit to act as a fail-safe bounding box for the canvas.

## 2. Technical Decisions & Dependencies
* **Target File:** `src/plugins/chord-notator/components/NotationCanvas.tsx`
* **Logic Specification:**
  * **Treble:** If `maxTrebleStep >= 25` (Canvas overflow imminent), apply `15ma` unconditionally. Else if `maxTrebleStep >= 18`, apply `8va` *only* if `minTrebleStep - 7 >= -2`.
  * **Bass:** If `minBassStep <= -22` (Canvas overflow imminent), apply `15mb` unconditionally. Else if `minBassStep <= -15`, apply `8vb` *only* if `maxBassStep + 7 <= 2`.
