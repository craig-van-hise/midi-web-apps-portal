# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
A state-bleeding bug occurs in the `NotationCanvas` when navigating the `StepSequencer` immediately after performing a Pitch-Class Set Rotation. The rotation correctly locks the `chordIdentityRef` to preserve enharmonic spellings during the turn, but timeline navigation fails to clear this lock when loading a new chord. Consequently, the new chord inherits the previous chord's spelling identity, mangling its output. The objective is to expand the `updateActiveNotes` Context API to accept a `clearIdentity` flag, explicitly drop the identity lock during timeline navigation, and apply the same safety drop to Undo/Redo history actions.

## 2. Technical Decisions & Dependencies
* **Target Files:**
  * `src/plugins/chord-notator/midi/MIDIProvider.tsx`
  * `src/plugins/chord-notator/components/NotationCanvas.tsx`
  * `src/plugins/chord-notator/components/StepSequencer.tsx`
* **Logic Specification:**
  * Update `MidiContextType` to allow `updateActiveNotes(notes: any[], clearIdentity?: boolean)`.
  * Intercept `clearIdentity` inside the `MIDI_MESSAGE_RECEIVED` handler of `NotationCanvas` to forcibly set `chordIdentityRef.current.isActive = false`.
  * Update `StepSequencer` to pass `true` during pointer-down and arrow navigation.
