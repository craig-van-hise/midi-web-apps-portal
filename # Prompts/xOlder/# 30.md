# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The VV | WebApps Portal is currently experiencing a critical state management bug in the global title bar's MIDI device selector. When multiple MIDI devices are connected, the dropdown populates correctly, but user attempts to change the active device fail. The selection instantly reverts to the first device in the list.

**Root Cause:** The `useEffect` handling `navigator.requestMIDIAccess()` in `src/core/App.jsx` incorrectly includes `selectedInputId` in its dependency array. When a user selects a new dropdown option, `setSelectedInputId` triggers a re-render, forcing the `useEffect` to re-execute. This re-requests MIDI access and unconditionally forces `setSelectedInputId(inputs[0].id)`, actively fighting and overwriting the user's manual input. 

**Objective:** Decouple the Web MIDI initialization cycle from the user's selection state to allow deterministic, persistent hardware selection.

## 2. Technical Decisions & Dependencies
* **Dependency Array Pruning:** The `requestMIDIAccess` `useEffect` must have an empty dependency array `[]` so it only runs once on component mount.
* **Functional State Updates:** To prevent stale closures inside `access.onstatechange` (since `selectedInputId` will no longer be in the dependency array), the agent must utilize React's functional state update pattern: `setSelectedInputId(prev => ...)`.
* **Mount Safety:** Implement an `isMounted` flag cleanup pattern within the `useEffect` to prevent state updates if the component unmounts before the MIDI promise resolves.

## 3. Implementation Phases

### Phase 1: Decouple MIDI Initialization & State Sync — ✅ Completed
* **Objective:** Refactor the global MIDI initialization logic in `src/core/App.jsx` to respect manual port selection.
* **Instructions:**
    1.  Locate the `useEffect` block responsible for `navigator.requestMIDIAccess()` in `src/core/App.jsx`.
    2.  Change its dependency array from `[selectedInputId]` to `[]`.
    3.  Implement an `isMounted` boolean to wrap the `.then()` state setters to prevent memory leaks on unmount.
    4.  Update the initial assignment inside `.then()` to conditionally set the input *only* if one is not already selected:
        ```javascript
        setSelectedInputId((prev) => prev || (inputs.length > 0 ? inputs[0].id : ''));
        ```
    5.  Refactor the `access.onstatechange` callback to use a functional update to evaluate the existing state without relying on a stale closure:
        ```javascript
        access.onstatechange = () => {
          const updatedInputs = Array.from(access.inputs.values());
          setMidiInputs(updatedInputs);
          
          setSelectedInputId((prevId) => {
            if (updatedInputs.length > 0) {
              // Keep the current selection if it still exists in the updated list
              const stillExists = updatedInputs.some(i => i.id === prevId);
              return stillExists ? prevId : updatedInputs[0].id;
            }
            return '';
          });
        };
        ```
