# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The `NotationCanvas` layout engine's Crossover Inhibition rule is currently too loose, allowing shifted notes to invade the opposing clef's physical space (e.g., an `8vb` shifted G3 renders visually as a G4 inside the treble staff). The objective is to tighten the Crossover Inhibition constraints so that shifted notes remain cleanly partitioned. 
* A shifted bass chord must not exceed E4 (the bottom boundary of the treble staff).
* A shifted treble chord must not drop below A3 (the top boundary of the bass staff).

## 2. Technical Decisions & Dependencies
* **Target File:** `src/plugins/chord-notator/components/NotationCanvas.tsx`
* **Logic Specification:**
  * Middle C (C4) is step `0`.
  * **Treble Crossover Limit:** A down-shifted treble chord must not contain a note below step `-2` (A3).
  * **Bass Crossover Limit:** An up-shifted bass chord must not contain a note above step `2` (E4).
