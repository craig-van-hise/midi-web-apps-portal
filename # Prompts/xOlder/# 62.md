# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
**Objective:** Implement an intelligent, latency-free MIDI playback system for the `midi-tonnetz` plugin.
**Current State:** The plugin visually maps Pitch Classes (modulo 12) from incoming MIDI but does not properly route internal UI interactions (node clicks, transformations, "Play" button) back to the host Rompler.
**Outcome:** The plugin will track absolute MIDI notes in parallel with relative Pitch Classes. It will utilize a "Center of Gravity" algorithm for isolated node additions and a "Parsimonious Proximity" voice-leading algorithm for musical transformations, seamlessly bridging Pitch Class Space with Absolute Pitch Space.

## 2. Technical Decisions & Dependencies
* **State Architecture Split:** * `activePitchClasses` (Array of numbers `0-11`): Drives the visual grid.
  * `activeMidiNotesRef` (Mutable React `useRef` array of numbers `0-127`): Drives the audio and mathematical calculations.
* **Core Algorithms (Pseudo-code):**
  * **Center of Gravity (For Node Clicks):**
    ```javascript
    const calculateGravityNote = (targetPc, currentMidiNotes) => {
      if (currentMidiNotes.length === 0) return targetPc + 60; // Default to Octave 4
      const meanPitch = currentMidiNotes.reduce((a, b) => a + b, 0) / currentMidiNotes.length;
      
      let bestNote = targetPc;
      let minDiff = Infinity;
      for (let oct = 0; oct <= 10; oct++) {
        const candidate = targetPc + (oct * 12);
        const diff = Math.abs(candidate - meanPitch);
        if (diff < minDiff) { minDiff = diff; bestNote = candidate; }
      }
      return bestNote;
    };
    ```
  * **Parsimonious Proximity (For Transformations):**
    ```javascript
    const calculateVoiceLeading = (targetPcs, previousMidiNotes) => {
      if (previousMidiNotes.length === 0) return targetPcs.map(pc => pc + 60);
      
      return targetPcs.map(pc => {
        let bestNote = pc;
        let minDiff = Infinity;
        for (let oct = 0; oct <= 10; oct++) {
          const candidate = pc + (oct * 12);
          // Find the distance to the closest previous note
          for (const prevNote of previousMidiNotes) {
            const diff = Math.abs(candidate - prevNote);
            if (diff < minDiff) { minDiff = diff; bestNote = candidate; }
          }
        }
        return bestNote;
      });
    };
    ```

## 3. Implementation Phases

### Phase 1: State Upgrades & The "Play" Trigger — ✅ Completed
* **Objective:** Introduce absolute MIDI tracking and a direct playback method.
* **Instructions:**
  1. Add a new `activeMidiNotesRef` (`useRef<number[]>([])`) to `index.tsx` alongside the visual active node state.
  2. Modify the incoming MIDI listener (from the `midiBus`) to push/filter actual MIDI note numbers (`0-127`) to `activeMidiNotesRef.current` synchronously.
  3. Create a `triggerPlay()` function that reads `activeMidiNotesRef.current` and fires `onMidiOut([144, note, 100])` for each note, followed by a `setTimeout` of 500ms to fire `onMidiOut([128, note, 0])` (Note Off).

### Phase 2: Center of Gravity Integration (Node Clicks) — ✅ Completed
* **Objective:** Map UI node clicks to intelligent octaves.
* **Instructions:**
  1. Locate the `onNodeClick` handler in `index.tsx`.
  2. Implement the `calculateGravityNote` algorithm defined in Section 2.
  3. When a node is toggled **ON**: Execute the algorithm to find the specific absolute MIDI note. Push it to `activeMidiNotesRef.current`.
  4. Immediately trigger `onMidiOut` with Note On (144) for that calculated note, and set a timeout for Note Off (128) just like the Play trigger.
  5. When a node is toggled **OFF**: Find the active absolute note modulo 12 that matches the clicked PC, remove it from the ref, and send a Note Off.
* **TDD Checkpoint:** * **Test Case 1:** [Given grid is empty, When node 'C' is clicked, Assert `onMidiOut` fires for Note 60].

### Phase 3: Parsimonious Proximity Integration (Transformations) — 🔵 Active
* **Objective:** Ensure D-pad grid transformations adhere to physical voice-leading.
* **Instructions:**
