## 1. Root Cause Analysis
1. **Rolling Chords (Jitter):** `postMessage` forces MIDI events into the browser's IPC queue. Simultaneous chord notes are serialized and processed across multiple 2.9ms audio blocks, causing a "rolling" or arpeggiated sound. We must bypass the IPC queue entirely using a lock-free `SharedArrayBuffer` (SAB).
2. **SAB Memory Corruption:** Previous SAB attempts failed silently because `RingBuffer.js` used an 8-bit `Uint8Array`, but `RomplerWorklet.js` read it as a 32-bit `Int32Array`. Furthermore, an 8-bit array caps at 255, causing the pointers to overflow immediately. Both ends MUST be strictly aligned to a 4096-byte `Int32Array`.

  while (true) {
    const writePtr = Atomics.load(this.sabArray, 0);
    const readPtr = Atomics.load(this.sabArray, 1);
    if (writePtr === readPtr) break;

    const status = Atomics.load(this.sabArray, readPtr);
    const note = Atomics.load(this.sabArray, readPtr + 1);
    const velocity = Atomics.load(this.sabArray, readPtr + 2);

    let nextRead = readPtr + 3;
    if (nextRead > 1021) nextRead = 2;
    Atomics.store(this.sabArray, 1, nextRead);

    const command = status & 0xf0;
    if (command === 0x90 && velocity > 0) this.handleNoteOn(note, velocity);
    else if (command === 0x80 || (command === 0x90 && velocity === 0)) this.handleNoteOff(note);
  }
}
