# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The `chord-notator` plugin was ported into a Master Portal host. Two architectural bugs were introduced during the migration:
1. **State Entanglement:** Physical MIDI hardware inputs are being processed as virtual UI clicks, causing "Toggle" mode to incorrectly apply to hardware controllers.
2. **Redundant UI:** A legacy "Click to Start Audio Engine" modal overlay remains in the `NotationCanvas`, which is now obsolete as the host portal manages global audio initialization.

## 2. Technical Decisions & Architecture
* **MIDI Event Routing:** We will not touch the state logic in `NotationCanvas.tsx`. Instead, we will expose a new `dispatchPhysicalMidi` function from the `MIDIProvider` context. `index.jsx` will be refactored to route the `midiBus` events through this physical dispatcher (`isVirtual = false`), while the `<Keyboard />` component will continue using `dispatchVirtualMidi` (`isVirtual = true`).
* **Modal Stripping:** We will forcefully strip the `isAudioUnlocked` React state and its associated DOM nodes from `NotationCanvas.tsx` to rely strictly on the Host Portal's Audio Context.
