
# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
You are building the **VV | WebApps Portal** (`midi-web-apps-portal`, Version 1.0 MVP). This is a unified, zero-friction desktop browser environment to showcase, test, and demo standalone VV MIDI web applications. 

The portal wraps individual repository deployments into a single stateful dashboard, preserving global MIDI and Audio contexts across application switching. **Crucially, this is NOT a monorepo.** Child applications remain in their own GitHub repositories and are loaded into the Portal via an HTML `<iframe>`. The Portal acts as the master host, owning the Web MIDI API connection and the Tone.js audio engine, communicating with child apps entirely via `window.postMessage`.

## 2. Technical Decisions & Dependencies
* **Core Framework:** React 18 bootstrapped via Vite.
* **Styling:** Tailwind CSS. The UI is a strict 100vh layout with a DAW-inspired dark mode aesthetic (deep charcoals, off-blacks, vibrant accents).
* **Icons:** `lucide-react`.
* **State Management:** `zustand`.
* **Audio Engine:** Tone.js (Ported Rompler). 
* **MVP Exclusions (STRICTLY OUT OF SCOPE):** Do NOT implement Mobile Responsiveness (desktop only). Do NOT implement Global Transport (master clock/tempo sync). Do NOT implement App-to-App routing (apps operate in parallel isolation, routing directly to the Master Rompler).
* **Algorithms (MIDI to Frequency):** If raw frequency conversion is required for Tone.js, use standard tuning where $n$ is the MIDI note number:
  $$f = 440 \times 2^{(n - 69) / 12}$$

### Phase 2: Message Bus Protocol Implementation — ✅ Completed

* **Objective:** Establish the strictly typed `window.postMessage` system for cross-origin communication.
* **Instructions:**
1. Create a message bus controller/hook (`useMessageBus.js`).
2. **MANDATORY:** You must format all Downstream (Portal-to-App) and Upstream (App-to-Portal) payloads exactly as follows:



**Portal-to-App Commands (Downstream)**

```javascript
// 1. Raw hardware MIDI input sent to the app for processing
{ type: 'VV_MIDI_IN', payload: [status, data1, data2] }
// 2. Clear all internal app buffers/states
{ type: 'VV_PANIC' }
// 3. Toggle the app's local Info modal state
{ type: 'VV_TOGGLE_INFO' }
// 4. Toggle the app's local Settings modal state
{ type: 'VV_TOGGLE_SETTINGS' }
// 5. Tell the app to enter or exit Bypass/Pass-through mode
{ type: 'VV_TOGGLE_POWER', payload: { active: boolean } }

```

**App-to-Portal Commands (Upstream)**

```javascript
// 1. Processed MIDI data sent from the app to the Portal's Rompler
{ type: 'VV_MIDI_OUT', payload: [status, data1, data2] }

```
