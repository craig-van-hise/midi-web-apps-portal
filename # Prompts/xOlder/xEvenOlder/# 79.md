# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The Chord Notator's binary lookup table (`PCS_LUT.dat`) is returning incorrect chord strings (e.g., scale modes and binary IDs) instead of standard chord types. The root cause is a severe byte-offset misalignment in `binaryLut.ts`, which fails to respect the 64-byte fixed header specification defined during the `.dat` generation. The objective is to refactor the binary decoding logic to accurately extract values using the correct byte offsets.

## 2. Technical Decisions & Dependencies
* **Data Struct Alignment:** We will map the `DataView` offsets directly to the `PLUT` specification:
    * `decimal`: +0 (`uint32`)
    * `root_pc`: +4 (`uint8`)
    * `cardinality`: +5 (`uint8`)
    * `chord_type` index: +16 (`uint16`)
    * `base_triad` index: +20 (`uint16`)
* **Variable Array Parsing:** Instead of guessing interval counts based on the next row's offset, we will read the explicit array length counters located between bytes 44-51, and sequentially read the variable data starting precisely at byte 64.
