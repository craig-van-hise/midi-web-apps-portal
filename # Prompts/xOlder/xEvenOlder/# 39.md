# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
**Objective:** Eradicate Main Thread latency by gutting the Tone.js sample generators (`LoopedSampler`, `Tone.Sampler`) and replacing them with a custom `AudioWorkletProcessor`.
**Architecture:** The Worklet will autonomously poll the lock-free `SharedArrayBuffer` (SAB) for MIDI events and perform raw Digital Signal Processing (DSP) to generate polyphonic audio. The Worklet's output will be routed back into the existing Tone.js mixer chain (PanVol, Reverb) so the UI knobs remain functional.

## 2. Technical Decisions & DSP Algorithms
* **AudioWorklet DSP Math:**
  * The agent must write a 32-voice polyphonic synthesizer from scratch in standard JS (no libraries exist inside the Worklet).
  * **Pitch Shifting:** `playbackRate = Math.pow(2, (targetMidi - rootMidi) / 12)`. Phase pointer advances by `playbackRate` per frame.
  * **Linear Interpolation:** To read fractional indices in the `Float32Array`.
  * **Envelope State Machine:** 0=Idle, 1=Attack, 2=Decay, 3=Sustain, 4=Release. 
* **Buffer Transport:** The Main Thread will use `AudioContext.decodeAudioData` to convert the downloaded `.mp3`/`.ogg` files into raw `Float32Array` PCM data, and pass them to the Worklet via `port.postMessage`.

## 3. Implementation Phases

### Phase 1: Build the Custom DSP Worklet — ✅ Completed
* **Objective:** Create the isolated audio thread script.
* **Instructions:**
  1. Create a new file: `public/RomplerWorklet.js`. *(Note: It MUST be in the `public` folder so the browser can fetch it as a separate script).*
  2. Implement the DSP class extending `AudioWorkletProcessor`.
  3. **Memory Setup:**
     * Listen for `port.onmessage`. If `event.data.type === 'SAB'`, store the `Int32Array` view of the SharedArrayBuffer.
     * If `event.data.type === 'SAMPLES'`, store the incoming `Float32Array` buffers mapped by instrument/note.
  4. **The `process()` Loop:**
     * Before calculating the 128 audio frames, pull all pending MIDI messages from the SAB using `Atomics.load`. If Note-On, find a free voice (or steal the oldest) and set state to Attack. If Note-Off, set matching voice state to Release.
     * Loop through `0` to `127` (frames). For each frame, loop through the 32 voices. If active, calculate ADSR envelope multiplier, advance the phase pointer by `playbackRate`, apply linear interpolation to the buffer, and sum the result into the output channel.
  5. Call `registerProcessor('rompler-processor', RomplerWorklet);` at the bottom of the file.
