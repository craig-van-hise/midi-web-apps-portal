# Product Requirements Prompt (PRP) — 🔵 Active

## 1. Project Context & Objectives
The application requires UI layering fixes and enhanced user guidance. Currently, the global settings modal and context menus are rendering behind the lower keyboard component due to stacking context conflicts. Additionally, the transformation toolbar buttons lack descriptive tooltips, making it difficult for users to distinguish between chromatic, diatonic, and rotational transpositions. The objective is to force the modals to the absolute top of the stacking context and implement highly legible, animated tooltips using `@radix-ui/react-tooltip`.

## 2. Technical Decisions & Dependencies
* **Z-Index Strategy:** Upgrade the `z-index` classes in `SettingsModal.tsx` and `TransformationsContextMenus.tsx` from `z-50` / `z-[100]` to `z-[9999]` to definitively break out of local stacking contexts.
* **Tooltip Library:** Use `@radix-ui/react-tooltip` (already present in `package.json`).
* **Tooltip UI:** Tooltips will use `Tooltip.Portal` to render safely outside DOM constraints, ensuring they are not clipped by toolbar overflow. They will also use `z-[9999]` and a highly legible, dark-mode-styled typography.
