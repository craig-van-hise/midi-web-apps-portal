
# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives

The keyswitch illumination works on the initial press but fails on subsequent presses. The root cause is a React lifecycle issue: state updates triggered by the first keyswitch press cause context re-renders that desync or detach the event listeners inside `Piano88`. The objective is to eject the keyswitch tracker from the React component entirely, placing it at the module level so it operates exclusively on the stable DOM layer, immune to component lifecycles.

## 2. Technical Decisions & Dependencies

* **Payload Enrichment:** Modify `__MIDI_INTERCEPTOR` in `MIDIProvider.tsx` to include `midiNote: note` in the `APP_BUTTON_PRESS_ON` and `OFF` event details.
* **Module-Level Tracking:** Replace the React `useRef` inside `Piano88` with a standard JavaScript `Set` (`activeKeyswitchesTracker`) defined at the top of `Keyboard.tsx`.
* **Cleanup:** Delete the volatile `useEffect` event listeners from inside the `Piano88` component.

```typescript
// Absolute hardware tracking for keyswitches, immune to React lifecycle
const activeKeyswitchesTracker = new Set<number>();
if (typeof window !== 'undefined') {
    window.addEventListener('APP_BUTTON_PRESS_ON', (e: any) => {
        const note = e.detail?.midiNote;
        if (note !== undefined) {
            activeKeyswitchesTracker.add(note);
            updateKeyVisuals88(note, 'ks_active');
        }
    });
    window.addEventListener('APP_BUTTON_PRESS_OFF', (e: any) => {
        const note = e.detail?.midiNote;
        if (note !== undefined) {
            activeKeyswitchesTracker.delete(note);
            updateKeyVisuals88(note, '');
        }
    });
}

```
