# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The initial keyswitch visual implementation resulted in muddy colors, undersized badges, and DOM node stacking when mappings changed. Furthermore, active keyswitch presses are not flaring with their assigned semantic colors. The objective is to refine `Keyboard.tsx`: implement vibrant idle color washes using CSS `color-mix()`, increase badge size to `16px`, enforce strict DOM cleanup to prevent duplicate badges, and intercept the `updateKeyVisuals88` function to trigger a 100% opacity color flare when a keyswitch is pressed.

## 2. Technical Decisions & Dependencies
* **Color Mixing:** Use `color-mix(in srgb, ${hex} 30%, #ffffff)` for white keys and `color-mix(in srgb, ${hex} 35%, #444444)` for black keys to maintain vibrant, matched hues.
* **Badge Styling:** Set dimensions to `16px x 16px` with a `1.5px` solid black border, utilizing a `.ks-badge` class for easy DOM targeting/cleanup.
* **State Management:** Store the active keyswitch color in the DOM via `dataset.ksColor`. `updateKeyVisuals88` will read this attribute to determine the active flare and idle fallback colors.
