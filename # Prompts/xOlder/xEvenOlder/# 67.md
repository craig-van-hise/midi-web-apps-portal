# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
In the `chord-notator` plugin, applying programmatic rotation (inversion/voicing shifts) to complex chords causes the standard `PCS_LUT` algorithm to misidentify the new lowest note as a new root. This results in volatile, inconsistent chord naming and enharmonic misspellings across rotations (e.g., Gø9 morphing into Db+(#4,6)). 

**Objective:** Implement an "Identity Lock" state mechanism. When a programmatic rotation (`ROT`) is triggered, the system must retain the original chord's base name and spelling map, forcing the rotated notes to inherit these properties. The chord name should automatically format as a slash chord (e.g., `Gø9 / F`). Furthermore, if a locked chord is transposed (`SEMI`), the underlying identity and spelling map must transpose mathematically without breaking the lock (e.g., `Gø9 / F` shifted +1 becomes `Abø9 / Gb`).

## 2. Technical Decisions & Dependencies
* **Target Files:** `src/plugins/chord-notator/hooks/` (or state manager), `utils/chordSpeller.ts`, and the main component rendering the chord name/notation.
* **State Management (`IdentityContext`):** Introduce a state object (e.g., `chordIdentityRef`) that caches:
    1.  `isActive` (boolean)
    2.  `baseName` (e.g., "Gø9")
    3.  `rootPC` (e.g., 7 for G)
    4.  `spellingMap` (e.g., `{ 7: 'G', 10: 'Bb', 1: 'Db', 5: 'F', 9: 'A' }`)
* **Trigger Conditions:** * **Locking:** Updates normally when new physical MIDI input is received.
    * **Applying (ROT):** Sets `isActive = true`. Skips `PCS_LUT` and spells via `chordIdentityRef`.
    * **Transposing (SEMI):** If active, transposes the locked `rootPC`, `baseName` root, and `spellingMap` mathematically.
    * **Breaking:** New physical MIDI note-on/off events break the lock.

## 3. Implementation Phases

### Phase 1: Establish the Identity Cache — ✅ Completed
* **Objective:** Capture and store the spelling map and name of the chord before transformation occurs.
* **Instructions:**
    1.  Initialize a `useRef` or state for `chordIdentity` containing `isActive`, `baseName`, `rootPC`, and `spellingMap`.
    2.  Modify the standard spelling pipeline: whenever a chord is successfully evaluated via the normal Key/Scale override or `PCS_LUT` lookup, save the resulting name, root pitch class, and PC-to-string mapping into this ref. Ensure `isActive = false`.

### Phase 2: Intercept Rotation and Force Spelling — ✅ Completed
* **Objective:** Use the cached identity to spell the rotated notes and construct the slash chord name.
* **Instructions:**
    1.  In the `ROT` up/down handler, update the active notes array and immediately set `chordIdentity.isActive = true`.
    2.  In the rendering/spelling pipeline, add a conditional bypass at the top: `if (chordIdentity.isActive)`.
    3.  If active, map the new rotated MIDI notes to strings using *only* the `chordIdentity.spellingMap` (appending octaves mathematically: `Math.floor(note / 12) - 1`).
    4.  Determine the new lowest note's pitch class. If it equals `chordIdentity.rootPC`, output `baseName`. If it differs, output `${baseName} / ${spellingMap.get(lowestNotePC)}`.
* **TDD Checkpoint:** * **Test Case 1:** [Given cached identity of 'Gø9', When notes are rotated to [53, 57, 67, 70, 73] (F bass), Assert spelling output is `['F4', 'A4', 'G5', 'Bb5', 'Db6']`].

### Phase 3: Transposing Locked Identities — ✅ Completed
* **Objective:** Ensure mathematical transposition shifts the underlying identity without breaking the lock.
* **Instructions:**
    1.  Intercept the `SEMI` up/down handler. If `chordIdentity.isActive` is `true`, shift the `chordIdentity.rootPC` by the step delta: `(rootPC + delta + 12) % 12`.
    2.  Update the `baseName` string to reflect the new root (e.g., shifting G up a half-step changes the base name to Ab or G# depending on active key context/defaults).
    3.  Iterate over the cached `spellingMap`. Shift every key by the delta modulo 12, and update the string values to their transposed equivalents.
* **TDD Checkpoint:** * **Test Case 1:** [Given a locked identity of `Gø9 / F` (rootPC: 7), When a +1 SEMI event fires, Assert `rootPC` becomes 8, `baseName` becomes `Abø9` (or `G#ø9`), and the lowest note spelling maps to `Gb` (or `F#`)].
