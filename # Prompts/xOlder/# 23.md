## 1. Project Context & Objectives
The `KeySignatureSelector` needs a visual overhaul to match the application's modern, hardware-inspired design language. The goal is to transform the currently clunky, native browser dropdowns into a unified "Floating Pill" control using strict typography (`Jost`), minimal hover states, and layered shadows, all while retaining the existing data-binding logic to the `MidiContext`.

## 2. Technical Decisions & Architecture
* **Logic Preservation:** We will strictly maintain the current state handling (`keySignature`, `setKeySignature`) and the parsing logic (`currentRoot`, `currentScale`). No context or dataflow changes are permitted.
* **CSS / Tailwind Strategy:** * The outer container will act as the "Pill" using `rounded-full`, `shadow-lg`, and `bg-white dark:bg-[#111]`.
  * The native `<select>` tags will be visually neutralized using Tailwind's `appearance-none`, `bg-transparent`, and `outline-none`.
  * A synthetic minimal chevron (`‚ñæ`) will be absolutely positioned behind the select's click-box using `pointer-events-none` to fake a custom dropdown UI while retaining native mobile/desktop click behavior.
* **Typography:** The component will explicitly enforce `font-family: 'Jost', sans-serif` via inline styles to ensure the geometric aesthetic overrides any global resets.
