# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The project recently migrated to a custom `AudioWorklet` and `SharedArrayBuffer` (SAB) architecture to reduce latency. While latency dropped, this introduced severe DSP regressions: clipping, bad voice-stealing (clicks/pops), and erratic envelope swells. Furthermore, the removal of the Convolver reverb resulted in a dry, lifeless output, and the lack of automatic gain staging caused massive volume discrepancies between instruments. 

**Objective:** We are abandoning the custom SAB/Worklet architecture. We will revert to the battle-tested `Tone.js` architecture to regain perfect DSP math and automatic volume normalization. To maintain the new low-latency floor, we will specifically replace Tone's Convolution Reverb with a zero-latency Algorithmic Reverb (`Tone.Freeverb`). 

## 2. Technical Decisions & Dependencies
* **Audio Engine Replacement:** `src/core/rompler/engine.js` will be entirely overwritten using the provided `engine_legacy_reference.js` logic, but adapted for algorithmic reverb.
* **Reverb Substitution (CRITICAL):** Do **NOT** use `Tone.Reverb()`. You must use `Tone.Freeverb({ roomSize: 0.7, dampening: 4000 })`. This bypasses FFT block processing and eliminates delay compensation latency.
* **Decommissioning SAB:** `RomplerWorklet.js` and `RingBuffer.js` are now dead code and must be removed to prevent confusion. All MIDI events will fire synchronously on the main thread directly into `audioEngine.noteOn()`.
* **Lookahead Tuning:** Maintain `Tone.context.lookAhead = 0.002`. Audio triggers must use `Tone.now()` strictly.
* **Gain Map Preservation:** The specific gain normalizations from the legacy file (Electric Piano 2.0, Harp 2.5, etc.) must be hardcoded into the new engine initialization block.
