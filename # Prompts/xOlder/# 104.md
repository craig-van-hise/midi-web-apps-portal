# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The `NotationCanvas` Auto-Zoom feature needs structural and mathematical refinement. 
1. The CSS scaling is currently applied to the parent `grand-staff-system` container, improperly shrinking UI overlay cards (Key Signature, Chord Symbol). 
2. The trigger threshold (`110px`) is too aggressive, zooming the canvas on standard notes like A5 instead of reserving it strictly for extreme un-shiftable chords (like C1 + F6).
3. The zoom state persists when the canvas is cleared because the empty-state early return skips the scale reset logic.

## 2. Technical Decisions & Dependencies
* **Target File:** `src/plugins/chord-notator/components/NotationCanvas.tsx`
* **Architecture Change:** Introduce an inner `<div className="notation-zoom-layer">` to sandbox the CSS `transform` exclusively to the musical staves, barlines, and notes, leaving the UI cards isolated at `scale(1)`.
* **Math Adjustment:** Increase `TARGET_HALF_HEIGHT` to `135`. On a 320px canvas (160px half-height), this allows standard ledger lines to breathe but kicks in right before the physical bounds are breached.

       ```typescript
       // Auto-Zoom Calculation
       let maxAbsY = 0;
       allNotes.forEach(n => {
           if (Math.abs(n.y) > maxAbsY) maxAbsY = Math.abs(n.y);
       });
       labels.forEach(l => {
           const labelAbsY = Math.abs(l.y - l.offset);
           if (labelAbsY > maxAbsY) maxAbsY = labelAbsY;
       });

       // 30px padding for noteheads/stems
       const requiredHalfHeight = maxAbsY + 30; 
       const TARGET_HALF_HEIGHT = 135; // Relaxed threshold for 320px canvas

       if (requiredHalfHeight > TARGET_HALF_HEIGHT) {
           setStaffScale(TARGET_HALF_HEIGHT / requiredHalfHeight);
       } else {
           setStaffScale(1);
       }
       ```
