## 1. Project Context & Objectives
The VV | WebApps Portal is experiencing a "strummed" audio effect during polyphonic chord inputs. The root cause is Main Thread Blocking: heavy React UI rendering (Chord Notator, Matrices) triggered by the first note of a chord is locking the JS thread for 20-40ms. This traps subsequent MIDI events in the browser queue, causing them to trigger the audio engine late.

**Objective:** Implement strict UI Throttling using `useRef` and `lodash/throttle`. Audio execution must remain synchronous and instantaneous. React state updates must be decoupled via a throttled sync function to prevent the UI from blocking rapid sequential MIDI inputs.

## 2. Technical Decisions & Dependencies
* **Data Tracking:** Use a mutable `useRef` to track active notes and plugin data instantly without triggering React renders.
* **Throttling:** Use `lodash/throttle` to push the `useRef` data to the actual React `useState` at a maximum of ~30fps (32ms). 
* **Dependency:** Ensure `lodash` is installed. If not, the agent must install it.
