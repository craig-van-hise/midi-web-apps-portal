# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The `StepSequencer` component suffers from a synchronous event race condition during manual step selection. When switching selection from an active chord step to a blank step, the global `updateActiveNotes` broadcast triggers immediate event handling while `selectedStepRef.current` is stale. This causes the newly targeted blank array to destructively overwrite and erase the previously edited chord slot. The objective is to apply a synchronous ref mutation inside the pointer event handler to block cross-slot data bleeding.

## 2. Technical Decisions & Dependencies
* **Target File:** `src/plugins/chord-notator/components/StepSequencer.tsx`
* **Logic Specification:** Within the `onPointerDown` callback for the chord card, update `selectedStepRef.current` manually and synchronously *prior* to executing the `updateActiveNotes` call. This ensures any downstream synchronous listeners accurately identify the newly selected slot.

## 3. Implementation Phases
