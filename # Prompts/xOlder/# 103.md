# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The recent "Extreme Bypass" logic caused severe notation collisions because it forced global octavation shifts on wide-spanning chords, dragging upper notes down into the opposite clef. Furthermore, the primary product goal is to reserve guaranteed real estate at the top and bottom of the `NotationCanvas` (approx 50-60px) for upcoming UI features. The objective is to revert to strict crossover inhibition (to protect musical integrity) and implement an "Auto-Zoom" scaling factor. The layout engine will calculate the absolute pixel height of the rendered notation and dynamically shrink the Grand Staff container to guarantee it never breaches the reserved UI margins.

## 2. Technical Decisions & Dependencies
* **Target File:** `src/plugins/chord-notator/components/NotationCanvas.tsx`
* **Logic Specification:**
  * Define `TARGET_HALF_HEIGHT = 110`. Since the canvas is 320px tall (160px half-height), capping the notation at 110px guarantees a permanent ~50px safe margin at the top and bottom.
  * During `recalculateLayout`, evaluate the maximum absolute `y` position of all notes and ottava labels.
  * If `maxAbsY + padding > TARGET_HALF_HEIGHT`, calculate a proportional scale factor `< 1.0` and apply it to the staff container.

### Phase 2: Implement Auto-Zoom Scaling — ✅ Completed
* **Objective:** Calculate the maximum bounds and apply dynamic CSS scaling to protect the UI margins.
* **Instructions:**
    1. Still in `NotationCanvas.tsx`, define a new state near the top of the component:
       `const [staffScale, setStaffScale] = useState<number>(1);`
    2. Scroll to the very end of the `recalculateLayout` function (just before `setRenderedNotes([...allNotes]);`). Insert the bounding box calculator:
       ```typescript
       // Auto-Zoom Calculation
       let maxAbsY = 0;
       allNotes.forEach(n => {
           if (Math.abs(n.y) > maxAbsY) maxAbsY = Math.abs(n.y);
       });
       labels.forEach(l => {
           const labelAbsY = Math.abs(l.y - l.offset);
           if (labelAbsY > maxAbsY) maxAbsY = labelAbsY;
       });

       // 30px padding for noteheads/stems
       const requiredHalfHeight = maxAbsY + 30; 
       const TARGET_HALF_HEIGHT = 110; // Leaves 50px margins top and bottom

       if (requiredHalfHeight > TARGET_HALF_HEIGHT) {
           setStaffScale(TARGET_HALF_HEIGHT / requiredHalfHeight);
       } else {
           setStaffScale(1);
       }
       ```
    3. Scroll down to the JSX return statement. Locate the container div for the staves:
       `<div className="grand-staff-system relative w-[300px] h-full flex flex-col justify-center items-center">`
    4. Apply the scale variable via inline styles to this div:
       ```jsx
       <div 
         className="grand-staff-system relative w-[300px] h-full flex flex-col justify-center items-center"
         style={{ transform: `scale(${staffScale})`, transition: 'transform 0.15s ease-out' }}
       >
       ```
