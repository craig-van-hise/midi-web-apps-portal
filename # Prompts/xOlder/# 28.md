# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The previous attempt to fix the "viewport bleed" on the `pitch-class-matrix` plugin failed because `w-full` caps at the viewport width during horizontal scrolling, revealing the host portal's black background. Furthermore, the background color needs to be pure white to seamlessly blend the UI cards. The objective is to force a pure white background that aggressively expands to wrap all overflowing content.

## 2. Technical Decisions & Architecture
* **Scroll-Aware Background Expansion:** We will replace `w-full` with a combination of `w-full min-w-max min-h-screen`. The `min-w-max` utility forces the DOM node to be at least as wide as its maximum content width, ensuring the background color travels with the user as they scroll horizontally.
* **Pure White Flood:** The background will be explicitly set to `bg-white`, replacing the old `bg-[#f3f4f6]`.

## 3. Implementation Phases
