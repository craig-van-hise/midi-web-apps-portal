# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
We are implementing a semantic visual bridge between the `TransformationsToolbar` and the `Keyboard`. Mapped MIDI keyswitches must render a color-coded circular badge directly on the keyboard key, indicating the transformation type (via color) and direction (via arrow icon). The toolbar buttons must only reflect this color schema *if they are actively assigned to a keyswitch*. Unassigned buttons should remain neutral. Badges must be sized legibly, with white-key badges positioned significantly lower on the key than black-key badges.

## 2. Technical Decisions & Dependencies
* **Color Logic:** Integrate `TRANSFORM_SCHEMA` into the `TransformationsToolbar` and `Keyboard` logic.
* **Conditional Styling:** Toolbar buttons query `configs[id].midiNote` to determine if they should render with a semantic color or default to neutral black/gray.
* **Badge DOM Injection:** Use `updateKeyVisuals88` (the existing direct DOM manipulator) to inject badge elements into the keyboard.
* **Icons:** Utilize `lucide-react` icons (or raw SVGs matching them) within the keyboard badges.

## 3. Implementation Phases

### Phase 9: Conditional Toolbar Color Injection — ✅ Completed
* **Objective:** Color-code the toolbar buttons based on the `TRANSFORM_SCHEMA`, but *only* if they have an active MIDI mapping.
* **Instructions:**

### Phase 10: Keyboard Badge Rendering & Positioning — ✅ Completed
* **Objective:** Render large, conditionally positioned badge overlays on the `Piano88` component.
* **Instructions:**
    1. **Expose Mappings:** Ensure `useMidi()` exposes `buttonConfigs` so the keyboard knows which notes are mapped to which actions.
    2. **Update `Keyboard.tsx`:** * Add a new helper `updateKeyswitchBadge(note: number, config: ButtonConfig | null)`.
       * This function should create or update a `div` element attached to the key.
       * **Size:** Make the circular badge larger (e.g., `width: 24px; height: 24px;`).
       * **Positioning:** * If `isBlack`, position the badge near the bottom of the black key (`bottom: 4px`).
