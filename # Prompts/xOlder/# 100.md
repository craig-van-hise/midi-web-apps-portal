# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The `NotationCanvas` layout engine requires two structural patches for extreme registers:
1. **Label Rendering Fix:** The SMuFL font (Bravura) only renders the numeric octavation glyphs (`8`, `15`). We need to decouple the labels into a composite object (`{ glyph, suffix }`) so the layout engine can render the numeral musically and the suffix (`va`, `vb`, `ma`, `mb`) using standard italic text.
2. **Crossover Inhibition:** Global octavation shifts (`8va`, `15ma`, `8vb`, `15mb`) are currently pushing wide-spanning chords (e.g., C1 to B3) across the Grand Staff, creating severe ledger-line collisions. Shifts must be blocked if they push the opposite end of the chord into the opposing clef's space.

## 2. Technical Decisions & Dependencies
* **Target Files:**
  * `src/plugins/chord-notator/utils/notationMath.ts` (SMuFL Glyph Updates)
  * `src/plugins/chord-notator/components/NotationCanvas.tsx` (Layout Engine & JSX)
* **Logic Specification:**
  * **Treble Crossover Limit:** A down-shifted treble chord must not contain a note below step `-4` (F3).
  * **Bass Crossover Limit:** An up-shifted bass chord must not contain a note above step `4` (G4).
