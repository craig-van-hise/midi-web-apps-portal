# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
**Objective:** Eradicate reactive audio loops causing dropped polyphonic hardware notes, and implement true imperative, momentary MIDI triggers for UI transformations.
**Current State:** A `useEffect` in `index.tsx` is listening to `getOutputNotes()` and reactively overriding hardware MIDI input mid-chord. Furthermore, the UI navigation components lack discrete pointer up/down events, preventing true momentary sustain cutoffs.
**Outcome:** The reactive audio `useEffect` will be deleted. Hardware MIDI will purely manage its own state. UI components (`TonnetzGridContainer`, Nav Controller) will be refactored to emit discrete `onDown` and `onUp` events, which will imperatively drive the voice-leading math and MIDI outputs.

## 2. Technical Decisions & Dependencies
* **Core Rule:** Audio generation (`onMidiOut`) must **never** occur inside a `useEffect` responding to state changes. It must only occur inside direct event handlers (`handleMidi`, `onPointerDown`, `onPointerUp`).
* **Hardware Isolation:** `handleMidi` will push/remove notes from `activeMidiNotesRef` but will NOT trigger any voice-leading calculations.
* **Component Prop Contract:** `TonnetzGridContainer` must be updated to replace `onDirectionalTrigger` with `onDirectionalDown(dir)` and `onDirectionalUp()`.
