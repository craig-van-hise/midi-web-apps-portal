# Product Requirements Prompt (PRP) — 🔵 Active

## 1. Project Context & Objectives
The VV | WebApps Portal is experiencing critical polyphonic MIDI degradation (hung notes) during rapid chord releases or when keys are held down (flooding the system with Aftertouch/Active Sensing). The root cause is that `App.jsx` currently uses React `useState` (`globalMidiData`) to pass rapid sequential MIDI messages down to plugins. React 18's state batching overwrites synchronous micro-bursts of MIDI events, destroying crucial Note Off messages before they render.

**Core Directive:** Audio/MIDI processing is priority zero; UI rendering is secondary. 
**Objective:** Refactor the host-to-plugin pipeline to use a Ref-Based Event Bus (`EventTarget`). This completely bypasses React's render cycle for MIDI data transmission, ensuring plugins receive every single hardware event instantly and synchronously.

## 2. Technical Decisions & Dependencies
* **Architecture:** `App.jsx` will instantiate a native `EventTarget` stored in a `useRef`. 
* **Routing:** The `handleMidiMessage` listener will dispatch a custom `CustomEvent` containing the raw MIDI array directly to this bus.
* **Plugin Interface:** The `midiIn` array prop will be replaced by a `midiBus` prop. Plugins will `addEventListener` directly to this bus to process data instantaneously.

## 3. Implementation Phases

### Phase 1: Implement the Event Bus in `App.jsx` — ✅ Completed
* **Objective:** Remove the batched React state and establish the synchronous Event Bus.
* **Instructions:**
    1. In `src/core/App.jsx`, remove: `const [globalMidiData, setGlobalMidiData] = useState(null);`
    2. Initialize the bus: `const midiBusRef = useRef(new EventTarget());`
    3. Update the `handleMidiMessage` function inside the Web MIDI `useEffect`:
       ```javascript
       const handleMidiMessage = (message) => {
         const data = Array.from(message.data);
         if (isPowerActive) {
           const customEvent = new CustomEvent('midi', { detail: data });
           midiBusRef.current.dispatchEvent(customEvent);
         }
       };
       ```
    4. Update the `<ActivePlugin />` props: Remove `midiIn` and add `midiBus={midiBusRef.current}`.

### Phase 2: Refactor Plugin Interface to Consume the Bus — ✅ Completed
* **Objective:** Update the baseline headless plugin architecture to listen to the bus instead of React props.
* **Instructions:**
    1. Open `src/plugins/DummyPlugin.jsx` (and any other active plugins like `chord-notator` or `pitch-class-matrix` that are suffering from the bug).
    2. Update the prop signature to accept `midiBus` instead of `midiIn`.
    3. Replace the `useEffect` listening to `midiIn` with an Event Listener:
       ```javascript
       useEffect(() => {
         if (!midiBus || isBypassed) return;
         
         const handleMidiEvent = (e) => {
           const data = e.detail;
           processMidi(data); // Instantly handle the logic
         };

         midiBus.addEventListener('midi', handleMidiEvent);
         return () => midiBus.removeEventListener('midi', handleMidiEvent);
       }, [midiBus, isBypassed]);
       ```
* **TDD Checkpoint:**
