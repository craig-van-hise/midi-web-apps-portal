## 2. Technical Architectural Truths (To be injected into Docs)
The agent must use the following facts to rewrite the documentation:
1. **Audio Engine**: The engine (`src/core/rompler/engine.js`) is now a hybrid `Tone.js` and `smplr` setup. It runs natively on the main thread but achieves low latency by keeping `Tone.context.lookAhead = 0.002` and strictly decoupling React UI renders (throttled) from audio triggers (synchronous).
2. **The Reverb Latency Trap**: The primary source of historical latency was `Tone.Reverb` (a `ConvolverNode`), which forces FFT block processing and delay-compensates the entire audio graph. This was replaced with `Tone.Freeverb` (an algorithmic Schroeder reverberator) to maintain zero block-processing delay.
3. **Parallel Send/Return Bus**: The audio graph is no longer a linear insert chain. To preserve spatial panning of the dry signal, the engine uses a parallel Aux Send architecture: `PanVol -> Destination` (Dry) AND `PanVol -> ReverbSend(Gain) -> Freeverb -> Destination` (Wet).
4. **MIDI Pitch Typecasting**: A critical bug where Tone.js instruments played in minor thirds (because it interpreted raw MIDI integers as Hertz ratios) was fixed. The engine now strictly typecasts incoming MIDI integers to Scientific Pitch Notation strings (`"C4"`) using `Tone.Frequency(note, "midi").toNote()` before passing them to `Tone.Sampler`.
5. **Decommissioned Tech**: All references to `SharedArrayBuffer`, lock-free IPC, `RomplerWorklet.js`, and `RingBuffer.js` must be permanently purged from the docs.
