# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The Radix tooltips attached to the interactive transformation buttons are causing visual clutter and obstructing rapid clicking. The objective is to remove the tooltips from the individual arrow and action buttons, and instead bind them exclusively to the static category labels (e.g., "SEMI", "KEY", "ROT") situated above the arrow pairs. The tooltips must render above the text.

## 2. Technical Decisions & Dependencies
* **Tooltip Target:** Shift the `<Tooltip.Trigger asChild>` wrapper from `renderArrowBtn` / `renderActionBtn` to the `<span>` element inside `renderArrowPair`.
* **Tooltip Positioning:** Enforce `side="top"` on the `<Tooltip.Content>` to ensure the tooltip window appears above the label.
* **Cursor UX:** Add `cursor-help` to the label `<span>` so the user has a visual affordance that the text is hoverable for more info.
