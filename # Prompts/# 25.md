# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives

The `NotationCanvas` needs structural alignment to perfectly mirror the physical footprint of the 88-key `Keyboard` component below it. The canvas must become a locked-width, styled card. Additionally, the floating "Chord Symbol" must be upgraded from plain text into a matching "Pill" UI element that displays a fallback dash (`"-"`) when no chord is active, achieving perfect bilateral symmetry with the Key Settings pill.

## 2. Technical Decisions & Architecture

* **Strict Dimensional Locking:** The `NotationCanvas` container will be explicitly locked to `w-[936px]` (matching the piano keyboard's width constraint) instead of `w-full`.
* **Symmetrical Pill UI:** The Chord Symbol will be wrapped in a container that perfectly mimics the CSS box-model and shadow profile of the `KeySignatureSelector` pill.
* **Layout Anchoring:** The `grand-staff-system` div will remain the central anchor. The left/right absolute positioning (`right-[100%]` and `left-[100%]`) currently in place works perfectly for aligning the pills to the outer edges of the staff brackets.
