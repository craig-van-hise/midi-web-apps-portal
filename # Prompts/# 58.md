# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The recent audio engine update successfully restored tuning and low-latency DSP. However, the reverb is currently configured as a series "insert" effect (`panVol.chain(reverb, Destination)`). This destroys spatial panning because the dry signal and reverb tail are panned together. 

**Objective:** Refactor the audio routing in `engine.js` from an insert architecture to a parallel Send/Return architecture. Additionally, update the default application state so that Reverb initializes at `0.1` and Release initializes at `1.0`.

## 2. Technical Decisions & Dependencies
* **Parallel Routing Architecture:**
  * **Dry Path:** `panVol` -> `Tone.Destination`
  * **Send Bus:** `panVol` -> `reverbSend (Tone.Gain)` -> `reverb (Tone.Freeverb)` -> `Tone.Destination`
* **Freeverb Wetness:** Because the reverb is now on an aux return, `this.reverb.wet.value` MUST be hardcoded to `1` (100% wet). The UI reverb slider will now control the `reverbSend` gain node, not the reverb's wet/dry mix.
* **Target Files:** * `src/core/rompler/engine.js`
  * `src/core/rompler/MasterRompler.jsx` (or wherever UI defaults are stored)

## 3. Implementation Phases

### Phase 1: Re-architect Audio Engine Routing — ✅ Completed
* **Objective:** Implement the Send/Return bus in `engine.js` and enforce the default Release envelope.
* **Instructions:**
  1. Open `src/core/rompler/engine.js`.
  2. In the `init()` method, create the send node: `this.reverbSend = new Tone.Gain(0.1);`.
  3. Ensure `this.reverb` is initialized with `this.reverb.wet.value = 1;`.
  4. Disconnect the old chain and establish the new parallel routing:
     ```javascript
     this.panVol.connect(Tone.Destination); // Dry Path
     this.panVol.connect(this.reverbSend);  // Post-Pan Aux Send
     this.reverbSend.connect(this.reverb);
     this.reverb.connect(Tone.Destination); // Wet Return
     ```
  5. Update the `setReverbWet(wet)` method to adjust `this.reverbSend.gain.value = wet` instead of `this.reverb.wet.value`.
  6. In `loadInstrument()`, explicitly set `release: 1.0` in the configuration object when instantiating `new Tone.Sampler({ ... })`.
