
# Product Requirements Prompt (PRP) — 🔵 Active

## 1. Project Context & Objectives
**Context:** Based on the source files, the previous implementation caused two severe UI bugs:
1. **The Interleaving Bug:** The Drawer slipped between the piano keys (`z-[40]`) and the Spelled Notes Strip because the Keyboard Card wrapper failed to create an isolated stacking context.
2. **The Floating Gap:** `top-full` correctly anchored the Drawer to the bottom of the card, but an internal margin inside the `TransformationsToolbar` component pushed the visible card down into space.
**Objective:** Implement a strictly isolated DOM hierarchy. Strip all internal top-margins from the drawer contents so `y: 0` is perfectly flush.

## 2. Technical Decisions & Dependencies
* **Stacking Context (`isolate`):** We must wrap both elements in a `relative isolate` container. The Keyboard Card gets `z-20`. The Drawer gets `z-10`. This completely prevents the Drawer from interacting with the `z-[40]` piano keys inside the card.
* **Anchor Math:** `absolute top-full` guarantees the Drawer's bounding box starts exactly where the Keyboard Card ends. If there is a gap, it is strictly due to padding/margin inside the Drawer's children, which must be zeroed out.
