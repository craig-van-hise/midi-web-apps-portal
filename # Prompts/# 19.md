# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The VV | WebApps Portal is suffering from a "strummed chord" latency effect. Polyphonic MIDI messages are arriving in the browser's event queue, but synchronous React state updates (UI rendering) triggered by the first note are blocking the main JS thread. This delays the subsequent `midimessage` events in the queue by 20-50ms each, causing sequential audio playback and massive perceived latency.

**Objective:** Implement strict Audio/UI Thread Decoupling. Audio commands must be executed synchronously and instantaneously. All React state updates (`setState`) triggered by MIDI events must be deferred using `requestAnimationFrame` to yield the main thread and allow the browser to empty the MIDI event queue.

## 2. Technical Decisions & Dependencies
* **Pattern:** The "Audio-First, Paint-Second" pattern.
* **Mechanism:** Native `requestAnimationFrame()`. By wrapping React state updates in `rAF`, we instruct the browser to process all pending `midimessage` I/O tasks immediately and batch the UI renders right before the next screen repaint.

## 3. Implementation Phases

### Phase 1: Decouple Master Rompler Output — ✅ Completed
* **Objective:** Ensure the portal's ingestion of processed MIDI data triggers audio instantly without waiting for the VU meter or note UI to render.
* **Instructions:**
    1. In `src/core/App.jsx`, locate `handleRomplerOutput`.
    2. Split the logic into two distinct blocks. Block 1 executes the audio methods (`window.playNoteOn` / `playNoteOff`) immediately.
    3. Block 2 wraps the React state update (`setActiveNotes`) inside a `requestAnimationFrame`.
    *Example Architecture:*
    ```javascript
    const handleRomplerOutput = useCallback((midiData) => {
      if (!midiData || midiData.length === 0) return;
      const [status, note, velocity] = midiData;
      const command = status & 0xf0;

      // 1. SACRED AUDIO EXECUTION (Synchronous)
      if (command === 0x90 && velocity > 0) {
        if (window.playNoteOn) window.playNoteOn(note, velocity);
      } else if (command === 0x80 || (command === 0x90 && velocity === 0)) {
        if (window.playNoteOff) window.playNoteOff(note);
      }

      // 2. DEFERRED UI RENDER (Asynchronous)
      requestAnimationFrame(() => {
        if (command === 0x90 && velocity > 0) {
          setActiveNotes((prev) => {
            if (prev.some((n) => n.note === note)) return prev;
            return [...prev, { note, velocity, time: Date.now() }];
          });
        } else if (command === 0x80 || (command === 0x90 && velocity === 0)) {
          setActiveNotes((prev) => prev.filter((n) => n.note !== note));
        }
      });
    }, []);
    ```
