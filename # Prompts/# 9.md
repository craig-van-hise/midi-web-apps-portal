**Execution Strategy: Sequential Batch Processing**
You must process these one by one to avoid context bleed. For each of the following targets, execute the "Standard Porting Protocol" below:

**The Standard Porting Protocol (For Each App):**

1. **Dependencies:** Check the app's `package.json` for unique dependencies and install them in the Portal.
2. **Ingestion:** Copy the `components`, `utils`, `lib`, and `hooks` folders into the new plugin directory. Copy any unique assets into the Portal's `public` folder.
3. **The Headless Refactor:** Create `index.jsx` in the plugin folder.
* It MUST accept the standard props: `({ midiIn, onMidiOut, isBypassed, showInfo, showSettings, triggerPanic })`.
* Strip out the `<Header>`, Title Bars, Logos, and native Modal trigger buttons.
* Strip out `<MidiInputSelector>` and `<MidiOutputSelector>`.
* Remove all `navigator.requestMIDIAccess()` logic and standalone Tone.js instances.
* Wire the core logic to the props (e.g., incoming data from `midiIn`, outgoing data to `onMidiOut`).


4. **Integration:** Update `src/core/App.jsx` and `appRegistry.js` to import and render the new plugin dynamically.
5. **Testing:** Run `npx vitest run` to ensure no imports are broken before moving to the next app in the list.
