# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
**Objective:** Implement an intelligent, dual-mode MIDI playback system for the `midi-tonnetz` plugin that respects physical MIDI input as an absolute reset and provides parallel voice-leading for transformations.
**Current State:** The plugin visually maps Pitch Classes but does not properly route audio or track absolute octaves.
**Outcome:** The plugin will track absolute MIDI notes. Hardware MIDI input will hard-reset the tracking state. UI transformations will utilize an "Index-Mapped" (Parallel) voice-leading algorithm by default, with an option to toggle to a "Parsimonious" (Common Tone) algorithm.

## 2. Technical Decisions & Dependencies
* **State Architecture Split:** * `activePitchClasses` (Array of numbers `0-11`): Drives the visual grid.
  * `activeMidiNotesRef` (Mutable React `useRef` array of numbers `0-127`): Drives the audio and mathematical anchor.
* **Core Algorithms (Pseudo-code):**
  * **Hardware Override (Absolute Reset):**
    ```javascript
    // Inside the midiBus listener
    if (isHardwareNoteOn) {
      // Do NOT apply gravity or voice leading. 
      // Push the raw note to the ref to reset the anchor.
      activeMidiNotesRef.current.push(rawMidiNote); 
    }
    ```
  * **Parallel Shift (Index-Mapped Voice Leading):**
    ```javascript
    const calculateParallelShift = (targetPcs, previousMidiNotes) => {
      if (previousMidiNotes.length !== targetPcs.length) {
          return calculateGravityNoteForMismatch(targetPcs, previousMidiNotes);
      }
      const sortedPrev = [...previousMidiNotes].sort((a, b) => a - b);
      const sortedPcs = [...targetPcs].sort((a, b) => a - b);

      return sortedPcs.map((pc, index) => {
        const prevNote = sortedPrev[index];
        let bestNote = pc;
        let minDiff = Infinity;
        for (let oct = -2; oct <= 10; oct++) { 
          const candidate = pc + (oct * 12);
          const diff = Math.abs(candidate - prevNote);
          if (diff < minDiff) { minDiff = diff; bestNote = candidate; }
        }
        return bestNote;
      });
    };
    ```

## 3. Implementation Phases

### Phase 1: Hardware MIDI Override & Play Trigger — ✅ Completed
* **Objective:** Ensure physical MIDI keys reset the octave range and implement the UI Play trigger.
* **Instructions:**
  1. Add `activeMidiNotesRef` (`useRef<number[]>([])`) to `index.tsx`.
  2. Modify the `midiBus` listener. When physical MIDI Notes arrive, they must bypass all calculation logic and directly overwrite/append to `activeMidiNotesRef.current` using their raw absolute note numbers (`0-127`).
  3. Create a `triggerPlay()` function that reads `activeMidiNotesRef.current` and fires `onMidiOut([144, note, 100])` for each note, followed by a 500ms timeout for Note Off (`[128, note, 0]`).
  4. Wire `triggerPlay` to the Nav Controller's `<Play />` button.

### Phase 2: Dual-Mode Transformation Voice Leading — 🔵 Active
* **Objective:** Ensure D-pad grid transformations adhere to parallel motion by default.
* **Instructions:**

  2. Implement the `calculateParallelShift` algorithm (Index-Mapped) defined in Section 2.
