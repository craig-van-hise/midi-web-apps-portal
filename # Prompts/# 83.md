# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The `SettingsModal` and the `TransformationsContextMenus` are being sliced by the Keyboard component because they are trapped inside lower `z-index` localized stacking contexts (`z-10` vs `z-20`). Furthermore, the transformation toolbar buttons require descriptive, highly legible tooltips. The objective is to use `ReactDOM.createPortal` to mount the modals directly to the DOM body to escape the layout trap, and to implement animated tooltips using `@radix-ui/react-tooltip`.

## 2. Technical Decisions & Dependencies
* **Z-Index Fix:** We will use `createPortal` from `react-dom` to mount `SettingsModal`, `ArrowContextMenu`, and `GlobalContextMenu` directly to `document.body`.
* **Tooltip Implementation:** We will use `@radix-ui/react-tooltip`, utilizing `<Tooltip.Portal>` to ensure the tooltips also escape the drawer's stacking context.
