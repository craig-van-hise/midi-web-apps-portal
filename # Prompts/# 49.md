## 1. Root Cause Analysis
1. **Dead Notes:** `RomplerWorklet.js` swallows repeated notes because the `handleNoteOn` method explicitly `return`s if a note is already active, rather than re-triggering the envelope.
2. **Latency:** The `postMessage` pipeline is bound to the Main Thread's IPC queue. When React renders, the event loop clogs, delaying the `postMessage` dispatch and maintaining a ~25ms latency floor. We must re-enable `SharedArrayBuffer` by applying COOP/COEP headers to the Vite server, which allows us to bypass the IPC queue entirely.
