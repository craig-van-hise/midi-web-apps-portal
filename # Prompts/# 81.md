# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The transformation directional arrows (Up/Down) in the `TransformationsToolbar` currently play audio with a hardcoded 500ms cutoff instead of sustaining the notes for the duration of the button press. The objective is to remove this hardcoded timeout in `NotationCanvas.tsx` and introduce an `APP_TRANSFORM_OFF` event, ensuring the directional arrows sustain and release notes exactly like the standard "Play" button.

## 2. Technical Decisions & Dependencies
* **Event Architecture:** `NotationCanvas` already has a perfect `handlePlayOff` function that releases whatever notes are currently selected. We will reuse this function by binding it to a new `APP_TRANSFORM_OFF` window event.
* **Canvas Refactoring:** Strip out the `setTimeout` wrappers inside the three transformation functions.
* **Parent Hookup:** The parent component that orchestrates `<TransformationsToolbar>` must be updated to dispatch `APP_TRANSFORM_OFF` on `pointerup`/`pointerleave`.
