# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
The `monitor` plugin is suffering from severe CSS layout blowouts when loaded inside the Master Portal. The right console (Continuous Data Stream) clips off the right side of the screen, and the piano keyboard clips off the bottom. The objective is to strip away viewport-relative units (`vw`/`vh`/`screen`) and replace them with standard responsive Flexbox constraints (`w-full`, `h-full`, `min-w-0`) so the plugin scales fluidly within the portal's designated workspace.

## 2. Technical Decisions & Architecture
* **Portal-Aware Dimensions:** The root container will drop `w-screen h-screen` in favor of `w-full h-full`, allowing the parent portal to dictate the maximum bounding box.
* **Strict Ledger Constriction:** The data table container will drop its inline `max-content` style. We will enforce a responsive fixed width (`w-[320px] lg:w-[450px]`), ensuring the Graph container always has space to shrink into.
* **Flex Container Lockdown:** The main `<main>` wrapper will drop `overflow-x-auto` in favor of `overflow-hidden`, forcing child elements to trigger their own internal scrolling rather than blowing out the parent horizontally.
