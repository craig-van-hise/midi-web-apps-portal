# Product Requirements Prompt (PRP) — Transposition Freeze Armor & Bug Trap ✅ Completed

## 1. Project Context & Objectives
The application experiences a catastrophic UI freeze (infinite loop) when a user repeatedly transposes a chord. Repeated transpositions are eventually causing a note object's properties (specifically `finalStep` or `stepOffset`) to evaluate to `NaN` or `undefined`. When this invalid data hits the `while (!placed)` collision-detection loops in the notation layout engines, the loop increments infinitely. We need to deploy hard circuit breakers to stop the freezing, sanitize the input, and log the corrupted data so the root transposition math bug can be tracked down.

## 2. Technical Decisions & Dependencies
* **Target Files:** `src/plugins/chord-notator/components/StepSequencer.tsx` AND `src/plugins/chord-notator/components/NotationCanvas.tsx` (and `notationMath.ts` if layout loops live there).
* **Circuit Breakers:** Every `while (!placed)` loop must be forcefully capped at 50 iterations.
* **Data Sanitization:** We will use `Number.isFinite()` to catch invalid steps and fallback to `0` to allow the render to complete safely.
* **Bug Trapping:** Inject `console.warn()` when invalid data is caught to expose the transpose math failure.

## 3. Implementation Phases

### Phase 1: Armoring the Mini Layout Engine (`StepSequencer.tsx`) — ✅ Completed
* **Objective:** Prevent `computeMiniLayout` from freezing when fed corrupted transpose data.
* **Instructions:**

  2. Replace the loop with this armored version:
     ```javascript
     let col = 0;
     let placed = false;
     let iter = 0;
     while (!placed && iter < 50) {
       iter++;
       if (!rightColumns[col]) rightColumns[col] = [];
       
       // Bug Trap & Sanitization
       let safeStep = note.finalStep;
       if (!Number.isFinite(safeStep)) {
         console.warn("BUG TRAP: Invalid finalStep detected in rightColumns layout", note);
         safeStep = 0; // Fallback to prevent math failure
       }
       
       if (!rightColumns[col].some(existingStep => Math.abs(existingStep - safeStep) <= 3)) {
         rightColumns[col].push(safeStep);
         placed = true;
       } else { col++; }
     }
     if (iter >= 50) console.error("CIRCUIT BREAKER TRIPPED: rightColumns layout loop exceeded 50 iterations.");
     ```
