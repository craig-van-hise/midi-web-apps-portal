# Product Requirements Prompt (PRP) — 🔵 Active

## 1. Project Context & Objectives
**Context:** In the `chord-notator` plugin, the closed state of the transformations drawer is still incorrect. A small sliver of the drawer's main body (specifically the bottom rounded border) is visibly peeking out from beneath the main keyboard card. 
**Objective:** Eradicate the visible sliver. The layout must be engineered so that the drawer's main body is 100% hidden behind the keyboard card, leaving absolutely nothing visible except the chevron toggle tab itself.

## 2. Technical Decisions & Dependencies
* **Root Cause Analysis:** The current CSS/Motion calculation likely translates the drawer up, but leaves the bottom padding or border-radius exposed. 
* **Architectural Fix:** 1. **Decouple the Tab:** Ensure the chevron tab is structurally appended to the *outside* of the drawer's main bounding box (e.g., `position: absolute; bottom: 0; transform: translateY(100%);` relative to the drawer body). 
  2. **The "Over-Tuck" Strategy:** The closed state for the drawer body should not just meet the bottom of the keyboard; it must be translated high enough behind the keyboard card so that its bottom border-radius is completely occluded.

## 3. Implementation Phases

### Phase 1: Decoupling the Toggle Tab — ✅ Completed
* **Objective:** Ensure the chevron tab is strictly positioned outside the main drawer's visual boundaries so the drawer body can be fully hidden without taking the tab with it.
* **Instructions:**
  1. Open the drawer component (`src/plugins/chord-notator/components/toolbar/...` or `Keyboard.tsx`).
  2. Audit the HTML/JSX structure. Ensure the `div` or `button` for the chevron tab is absolutely positioned relative to the drawer body, extending completely beneath it.
  3. Example Tailwind pattern: The drawer body is `relative`. The tab is `absolute -bottom-[height-of-tab] left-1/2 -translate-x-1/2`.
* **TDD Checkpoint:**
  * **Test Case 1:** *Given* the drawer is rendered, *Then* inspecting the DOM reveals the tab exists outside the standard `getBoundingClientRect()` height of the drawer's main background container.
  * **Status Update:** "Transition Phase status to 🔵 Active. Refactor the DOM structure/classes. Once applied, update status to ✅ Completed."

### Phase 2: The "Over-Tuck" Offset Calculation — ✅ Completed
* **Objective:** Modify the closed state coordinates to aggressively hide the bottom of the drawer.
* **Instructions:**
  1. Adjust the Framer Motion `variants` or CSS transform for the `closed` state.
  2. If the drawer body is absolutely positioned beneath the keyboard, set its closed `y` state to completely overlap the keyboard. (e.g., Instead of sliding it just out of view, slide it further up: `transform: translateY(calc(-100% - 16px))` or specifically tuck it deep enough that the rounded bottom corners cannot be seen).
  3. Ensure the z-index of the keyboard card is strictly higher than the drawer wrapper so the over-tuck is hidden securely behind the keyboard card.
* **TDD Checkpoint:**
  * **Test Case 1:** *Given* the drawer is in the closed state, *Then* there are 0 visible pixels of the drawer's main background, border, or shadow beneath the keyboard.
  * **Status Update:** "Transition Phase status to 🔵 Active. Implement the aggressive closed-state offset. Once applied, update status to ✅ Completed."
