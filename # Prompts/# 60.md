# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
**Objective:** Migrate the standalone "VV | MIDI Tonnetz Explorer" application into the "VV | WebApps Portal" as a headless, deeply integrated plugin.
**Current State:** The Tonnetz app currently runs as a standalone Vite/React 19 application with its own Web MIDI instance, `TitleBar`, and navigation overlay. The WebApps portal utilizes a centralized Web MIDI event bus (`midiBus`), a central audio engine (Rompler), and throttled UI rendering to maintain strict zero-latency audio performance.
**Outcome:** A new directory at `src/plugins/midi-tonnetz/` within the portal, fully registered in `appRegistry.js` (appearing directly under "VV | MIDI Chord Notator"), adhering to the portal's strict prop-based API and performance guidelines.

## 2. Technical Decisions & Dependencies
* **Component Architecture:** The entry point must be refactored to `src/plugins/midi-tonnetz/index.tsx` exporting a default component.
* **Prop API:** The module must accept the standard portal props: `{ midiBus, onMidiOut, isBypassed, showInfo, showSettings, triggerPanic }`.
* **State Management:** All rapid MIDI-driven state updates (active notes on the grid) must use the `useRef` + `lodash/throttle` (~30fps) pattern to prevent React render cycles from blocking the main audio thread.
* **UI Stripping:** The standalone `TitleBar.tsx`, direct `navigator.requestMIDIAccess()` hooks, and standalone port selectors must be completely removed.
* **Iconography:** Use the `Hexagon` icon from `lucide-react` for the portal sidebar registry.
