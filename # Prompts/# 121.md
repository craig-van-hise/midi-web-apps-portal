# Product Requirements Prompt (PRP) — Sequence Mapping UI Polish 🔘

## 1. Project Context & Objectives
The foundational logic for mapping `StepSequencer` chords to `Keyboard` keyswitches is already complete and active. This phase focuses purely on UI polish. We need to implement an instructional helper overlay when the user clicks "Map to Keys", and we need to add explicit bar numbers (1-8) to the sequencer grid so the physical keyswitch badges visually correspond to the correct bars.

## 2. Technical Decisions & Dependencies
* **Target File:** `src/plugins/chord-notator/components/StepSequencer.tsx`.
* **Listener Overlay:** We will utilize the existing `isListeningForMap` state (implemented in the previous phase) to conditionally render an absolute-positioned dark overlay covering the sequencer grid.
* **Z-Index Auditing:** Ensure the overlay sits above the staff lines and notes, but below any global popovers.

## 3. Implementation Phases

### Phase 1: Listener Mode Helper Overlay (`StepSequencer.tsx`) — ✅
* **Objective:** Guide the user to press a physical key when mapping mode is active.
* **Instructions:**
  1. Locate where the `isListeningForMap` state is evaluated in the `StepSequencer` render tree.
  2. Conditionally render an absolute-positioned overlay that completely covers the main `Sequencer Grid` area (the flex container holding the 8 bars). 
  3. Style the overlay with a dark translucent background (e.g., `bg-black/60 dark:bg-black/80`) and a backdrop blur to obscure the notation behind it.
  4. Center a highly visible text prompt inside the overlay: *"Press any key on your MIDI controller or virtual keyboard to assign the starting note for your sequence."*
  5. Add a "Cancel" button below the text that sets `isListeningForMap` to `false`.
* **TDD Checkpoint:** * **Test Case 1:** [Given `isListeningForMap` is true, When the UI renders, Assert the instructional overlay and Cancel button are visible].
  * **Status Update:** "✅ Completed"

### Phase 2: Sequencer Bar Numbers (`StepSequencer.tsx`) — 🔵
* **Objective:** Number the bars visually (1-8) so they map logically to the keyswitch badges.
* **Instructions:**
  1. Inside the `sequence.map((bar, idx) => ...)` loop rendering the 8 bars, locate the relative container for the `Mini Grand Staff System`.
  2. Inject an absolute-positioned `<span>` at the top left corner of each bar's staff area.
  3. The text should evaluate to `idx + 1` (so they read 1 through 8).
  4. Style it to be subtle but legible so it doesn't distract from the notation (e.g., `absolute top-1 left-2 text-[10px] font-bold text-black/30 dark:text-white/30`).
* **TDD Checkpoint:** * **Test Case 1:** [Given a rendered StepSequencer, Assert 8 distinct bar number elements exist containing text "1" through "8"].
