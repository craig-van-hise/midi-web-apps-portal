
# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
The 5-row collision algorithm implemented in the `Keyboard` component correctly identifies horizontal collisions, but its vertical row placement relies on percentage-based strings (`75%`, `45%`) that do not provide enough vertical clearance within a 30px container. This causes labels to visually overlap vertically (e.g., Row 1 and Row 2 crash into each other). The objective is to replace the percentage-based layout with strict pixel values, guaranteeing a 20px vertical gap between every row tier.

## 2. Technical Decisions & Dependencies
* **Target File:** `src/plugins/chord-notator/components/Keyboard.tsx`
* **Logic Specification:**
  * Redefine the 5 vertical tiers using absolute pixel values.
  * Priority 1 (Base): `20px` (Inside strip, lower)
  * Priority 2 (Top): `0px` (Inside strip, upper edge)
  * Priority 3 (Overhang Down): `40px` (Floating above piano keys)
  * Priority 4 (Overhang Up): `-20px` (Floating above the grey strip)
  * Priority 5 (Deep Overhang Down): `60px` (Deep into the piano keys)
  * This guarantees exactly 20px of vertical separation, easily accommodating the ~14px total height of the cards.
