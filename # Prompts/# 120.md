## 1. Project Context & Objectives
The goal is to enable users to map an 8-bar recorded chord sequence from the `StepSequencer` to physical/virtual keyswitches on the `Keyboard`. 
Users will select a root note (either by clicking a UI button and pressing a physical key, or by Option+Dragging the button directly to the virtual keyboard). The system will sequentially assign populated sequencer steps to contiguous chromatic keys starting from that root note. These keys will act as triggers: pressing them intercepts standard MIDI routing to play the entire recorded chord instead of a single note.

## 2. Technical Decisions & Dependencies
* **State Management (`MIDIProvider.tsx`):** We require a new state object `sequenceKeyswitches: Record<number, number>` mapping a physical `midiNote` (e.g., 36) to a sequencer `stepIndex` (0-7). 
* **Mapping Logic:** To keep things predictable, mapping uses **contiguous chromatic assignment**. If the user selects C1 (36) as the root, and steps 0, 1, and 3 have chords (step 2 is empty), C1 triggers step 0, C#1 triggers step 1, and D1 triggers step 3. 
* **Visual Badging (`Keyboard.tsx`):** We cannot fit full chord symbols on the keys. Instead, sequence keyswitches will be identified by a circular badge containing the sequence step number (1-8), colored with the sequencer's primary accent color (`#aa3bff`).
* **Drag & Drop Payload:** The drag event will use `e.dataTransfer.setData('application/json', JSON.stringify({ type: 'SEQUENCE_MAP_DRAG' }))`.
