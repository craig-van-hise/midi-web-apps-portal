# Product Requirements Prompt (PRP) — 🔘 Pending

## 1. Project Context & Objectives
During the recent audio engine reconstruction, a critical tuning bug was discovered affecting half of the Rompler instruments (Piano, Strings, Acoustic Guitar, Electric Bass). The user noted that playing an octave on the keyboard results in a minor third audio output. 

**Root Cause:** The routing logic in `src/core/rompler/engine.js` is passing raw MIDI integer values (e.g., 60, 72) directly to `Tone.Sampler.triggerAttack()`. Newer versions of Tone.js interpret raw integers as **Hertz**, not MIDI notes. (60Hz to 72Hz is a ratio of 1.2, which psychoacoustically equals a minor third, confirming the bug). The instruments using the `smplr` library were unaffected because that API expects MIDI integers natively.

**Objective:** Patch the `noteOn`, `releaseNote`, and internal triggering functions in `engine.js` to strictly typecast incoming numbers into Note Strings (e.g., "C4") before passing them to Tone.js instances.

## 2. Technical Decisions & Dependencies
* **Data Type Enforcement:** We must use `Tone.Frequency(note, "midi").toNote()` whenever interacting with `Tone.Sampler` or `LoopedSampler`. 
* **Target File:** `src/core/rompler/engine.js`
